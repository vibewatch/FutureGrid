## Headless-Chrome Smoke-Test Details

**Script:** `scripts/smoke-test.mjs`
- Serves static export in-process (Node HTTP server, port 3000)
- Spawns headless Chrome with `--no-zygote --headless=new` flags
- Loads all 9 routes sequentially; verifies each HTTP 200 + no error boundary
- On error: exits with code 1; on success: exits 0
- Process cleanup: async spawn + `process.kill(-pid)` to reap Chrome process group

**Critical Fixes:**
1. **Deadlock prevention:** Used async `spawn()` (not `spawnSync()`) — spawnSync blocks event loop, preventing static server from serving
2. **Process leak prevention:** Spawned Chrome with `{ detached: true }` + `--no-zygote` flag + `process.kill(-pid)` to reap entire process group

**Integration:**
- npm script: `"smoke": "node scripts/smoke-test.mjs"`
- CI job: added step `npm run smoke` after `npm run build` in test job
- Status: **GREEN** — all routes load without error boundary catch

## Gotchas Documented for Reuse

**Gotcha 1 — Deadlock:** Node HTTP server + spawnSync(chrome) in same process deadlocks. Fix: async spawn() with event loop unblocked.

**Gotcha 2 — Process leaks:** Chrome helper processes left alive after exit. Fix: spawn with detached:true + --no-zygote + process.kill(-pid) to reap process group.
